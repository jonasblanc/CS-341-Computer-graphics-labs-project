Vertex Normals

The starting point of this assignment are the functions in the file normal_computation.js. Here, you will have to compute the vertex normals weighted by the opening angles, which also requires computing the triangle normals. As you can see, this happens outside the shader so that we don’t have to compute these quantities for each pixel in the screen. For details on JavaScript syntax, you can refer to assignment 0’s handout.

When computing normals, you will handle arrays of length 3 which benefit from functions defined for vec3. A list of available methods can be found at this website. As an example, here is how you can compute a cross product between two arrays of length 3:

const a = [1., 0., 0.]
const b = [0., 1., 0.]
const c = vec3.cross([0., 0., 0.], a, b)
// c is equal to [0., 0., 1.]

Notice in the above that we need to provide an array as first input to vec3.cross(). More generally, for the function listed in the above website, the out argument should be replaced by [0., 0., 0.], which serves as a placeholder.

Efficient Ray-Mesh Intersections

To accelerate rendering, you have seen in class that spatial data structures such as KD-trees could be used. Instead of computing ray-triangle intersection for all triangles in the mesh, one could test if the ray intersect certain regions of the scene. Then, only triangles that belong to such intersected regions of space are candidates for a potential ray-triangle intersection. Testing intersections for those triangles only hence dramatically reduces the computational burden.

 

The building block for spatial data structures is the Axis Aligned Bounding Box (AABB), defined by a minimum corner and a maximum corner .
